# SCZ audit notes

## Fixed in this pass
- Chat/community image attach was labelled as avatar. It now uses separate `attach_image` / `image_ready` i18n keys.
- Feed/community posts now keep a local optimistic cache so a freshly sent post appears without waiting for relay consistency.
- Domain claim now checks the reputation gate and existing owner before publishing a claim.
- Wiki delete button used a back icon. It now uses the trash icon.
- Missing i18n keys for call labels and wiki controls were added.
- Tauri updater artifacts are enabled for release builds.
- Arch/Linux, APK, and GitHub release scripts/workflow were added.

## Still problematic
- Phase-0 domain ownership is still relay-derived and can race. Target model needs node-side indexed rules over signed events, then libp2p/Hypercore.
- Profile tag change can still publish a claim from the profile screen. It should reuse the same claim validation path as Domains.
- NIP-04 DM is legacy crypto. It is okay as Phase-0 compatibility, but should move to NIP-44 or the SCZ target encrypted-log model.
- Android build is debug APK for sideload testing. Production APK/AAB needs signing secrets and Android package review.
- AppImage/Windows updater requires real Tauri signing keys in GitHub secrets.
