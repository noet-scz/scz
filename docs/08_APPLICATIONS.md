# noet Applications: Как строить приложения на noet

> noet — это инфраструктура, не конечный продукт. Этот документ описывает, как реализовать популярные типы приложений на архитектуре noet без изменения ядра.

---

## Принципы построения приложений

1. **noet — транспорт и хранение, приложение — UX.** Приложение не меняет протокол, только представляет данные.
2. **Каждое приложение — клиент.** Множество приложений могут работать с одним сообществом.
3. **Данные принадлежат сообществу, не приложению.** Приложение — вью, данные — модель.
4. **Приложения могут быть специализированными.** Одно для форумов, другое для чатов, третье для стримов.

## 1. Discord-подобное приложение (Real-time Community)

### Что нужно от Discord
- Каналы (текстовые, голосовые)
- Роли и права доступа
- Модерация
- Боты/интеграции
- File sharing

### Реализация на noet

**Архитектура:**

```
Сообщество noet = Discord Server
├── Категории каналов → Namespace в сообществе
│   ├── #general → топик GossipSub "community/room/general"
│   ├── #random  → топик GossipSub "community/room/random"
│   └── #voice   → WebRTC room
├── Роли → записи в state CRDT (Yjs Map)
│   ├── @admin    → репутация + права в конституции
│   ├── @moderator → выборные участники
│   └── @everyone → базовые права
├── Права → проверка в клиенте + на узлах
├── Боты → приложения с API-доступом (NIP-46 подобие)
└── Файлы → IPFS CID в noet
```

**Поток сообщения:**

```
Пользователь пишет в #general
        ↓
Клиент создаёт noet типа "note"
        ↓
noet публикуется в GossipSub топик "community/room/general"
        ↓
Все подписчики получают noet в real-time
        ↓
noet сохраняется в Hypercore сообщества
        ↓
Новые участники при подключении читают Hypercore history
```

**Голосовые каналы:**

```
Пользователь заходит в #voice
        ↓
Клиент устанавливает WebRTC соединения с другими в канале
        ↓
< 8 участников: direct mesh P2P
        ↓
> 8 участников: election SFU node (или designated server)
        ↓
Аудио потоки: Opus через WebRTC data channels
```

**Что отличается от Discord:**

| Аспект | Discord | noet |
|---|---|---|
| Сервер | Discord контролирует | Сообщество контролирует |
| Данные | На серверах Discord | На узлах сообщества |
| Модерация | Discord Trust & Safety | Сообщество само модерирует |
| Боты | Discord API | Любой клиент может быть "ботом" |
| Голос | Discord серверы | P2P mesh + опциональный SFU |
| Форк | Невозможен | Всегда возможен |

## 2. Telegram-подобное приложение (Messaging)

### Что нужно от Telegram
- Быстрые сообщения
- Каналы (1-to-many)
- Группы
- Файлы
- Голосовые сообщения
- Боты

### Реализация на noet

**Личные сообщения (Direct Messages):**

```
Peer-to-peer шифрованные сообщения:
├── Каждый DM = отдельный зашифрованный Hypercore
├── Доступ только у двух участников (по их публичным ключам)
├── Синхронизация через libp2p direct connection
├── Offline: сообщения буферизуются на узлах (зашифрованные)
└── Forward secrecy: сессионные ключи ротируются
```

**Каналы:**

```
Канал = сообщество с правами:
├── Только автор может публиковать
├── Подписчики только читают
├── noet публикуется в GossipSub
├── Репликация через Hypercore (только автор пишет)
└── Подписчики синхронизируют read-only копию
```

**Группы:**

```
Группа = сообщество noet:
├── Множество писателей
├── GossipSub топик для real-time
├── Hypercore для истории
├── Администраторы через governance
└── Права через конституцию
```

**Голосовые сообщения:**

```
Голосовое сообщение = noet:
├── Контент: IPFS CID на .ogg opus файл
├── Метаданные: длительность, waveform preview
├── Проигрывание: потоковое через WebTorrent/IPFS
└── Оффлайн: кэширование в IndexedDB
```

**Что отличается от Telegram:**

| Аспект | Telegram | noet |
|---|---|---|
| Регистрация | Телефон | Ключевая пара |
| Серверы | Telegram DC | Распределённые узлы |
| E2EE | Только Secret Chats | Всё E2EE по умолчанию |
| Боты | Bot API | Любой клиент с API ключом |
| Цензура | Telegram может блокировать | Невозможна технически |

## 3. Reddit-подобное приложение (Forum)

### Что нужно от Reddit
- Subreddits (тематические сообщества)
- Голосование (upvote/downvote)
- Комментарии (дерево)
- Модерация subreddit
- Karma
- Cross-posting

### Реализация на noet

**Subreddit = Сообщество noet:**

```
Сообщество "programming":
├── Конституция: правила постинга, модерации
├── noet типа "thread" = Reddit post
├── noet типа "reply" = Reddit comment
├── Голосование: upvote/downvote как noet типа "reaction"
├── Karma: агрегация reputation в контексте сообщества
└── Модераторы: выборные через governance
```

**Дерево комментариев:**

```
thread-noet ("Какой язык программирования выбрать?")
├── reply-noet ("Go — отличный выбор для бэкенда")
│   ├── reply-noet ("Согласен, но как насчёт Rust?")
│   │   └── reply-noet ("Rust сложнее, но быстрее")
│   └── reply-noet ("Go проще в изучении")
├── reply-noet ("Python для старта")
└── reply-noet ("Зависит от задачи")
```

**Upvote/Downvote:**

```
reaction-noet:
├── target: <id thread или reply>
├── type: "upvote" или "downvote"
├── weight: функция от reputation голосующего
└── Агрегация: каждый клиент считает по своему алгоритму
```

**Модерация:**

```
Модераторское действие = noet:
├── type: "moderation_action"
├── action: "remove" | "lock" | "ban" | "warn"
├── target: <id noet или participant>
├── reason: ссылка на правило конституции
└── signature: подпись модератора
```

**Cross-posting:**

```
Кросс-пост = noet:
├── type: "crosspost"
├── original: <id noet в сообществе A>
├── target_community: <id сообщества B>
└── Контент: ссылка + цитата (не копия)
```

## 4. YouTube-подобное приложение (Video Platform)

### Что нужно от YouTube
- Загрузка видео
- Хранение
- Streaming
- Комментарии
- Подписки
- Рекомендации
- Монетизация

### Реализация на noet

**Загрузка и хранение:**

```
Видео = файл + метаданные:
├── Файл: загружается в IPFS (Kubo/Helia node)
├── CID: уникальный идентификатор видео
├── Метаданные: title, description, tags, thumbnail CID
├── noet: публикуется в сообществе ("My Video")
└── Thumbnail: отдельный файл в IPFS
```

**Streaming:**

```
Просмотр видео:
├── Клиент получает CID видео
├── WebTorrent: streaming через BitTorrent DHT
├── IPFS: fallback через Bitswap
├── Адаптивный битрейт: через HLS сегменты, каждый — отдельный торрент
└── Seeding: зрители раздают смотренные сегменты
```

**Комментарии:**

```
Комментарий к видео = reply-noet:
├── target: <id video-noet>
├── content: текст комментария
├── timestamp: временная метка видео (опционально)
└── Nested replies: как на Reddit
```

**Подписки:**

```
Подписка на канал:
├── Пользователь подписывается на noet автора
├── GossipSub: автоматическая доставка новых noet
├── Feed: агрегация подписок в хронологическом порядке
└── Локальный алгоритм сортировки (не центральный)
```

**Монетизация (опциональная):**

```
Монетизация = сообщество решает:
├── Подписки: платный доступ через крипто-токены сообщества
├── Донаты: Lightning Network / on-chain transfers
├── NFT (опционально): право собственности на редкий контент
└── Treasury: доходы идут в казну сообщества
```

**Рекомендации:**

```
Ключевое отличие от YouTube:
├── Нет центрального алгоритма
├── Каждый клиент реализует свой алгоритм
├── Пользователь выбирает/настраивает алгоритм
├── Open-source алгоритмы сообщества
└── Прозрачность: пользователь видит, почему контент рекомендован
```

## 5. Twitch-подобное приложение (Live Streaming)

### Что нужно от Twitch
- Live streaming
- Чат
- Подписки/донаты
- Записи стримов
- Клипы

### Реализация на noet

**Live streaming:**

```
Стример → noet:
├── Стример публикует stream-announcement noet
├── Видео: WebRTC ingest → сегментация
├── Каждый сегмент: торрент (WebTorrent)
├── Зрители: скачивают сегменты через P2P
└── Chat: GossipSub топик "stream/<id>/chat"
```

**Технические детали:**

```
Стриминговая pipeline:
┌─────────────┐    WebRTC    ┌─────────────┐   IPFS   ┌─────────────┐
│   OBS/      │ ───────────> │  noet Node   │ ───────> │  IPFS/      │
│   Encoder   │   (low-lat)  │ (segmenter) │  (store) │  WebTorrent │
└─────────────┘              └─────────────┘          └──────┬──────┘
                                                             │
                    ┌────────────────────────────────────────┘
                    │ P2P distribution
                    ↓
           ┌──────────────┐
           │   Viewers    │
           │ (WebTorrent) │
           └──────────────┘
```

**Задержка:**

| Этап | Задержка |
|---|---|
| Кодирование | 100-500ms |
| Сегментация | 1-5 сек (зависит от размера сегмента) |
| P2P распространение | 1-10 сек (зависит от числа пиров) |
| **Итого** | **3-15 секунд** (выше Twitch, но P2P) |

**Оптимизации задержки:**
- Короткие сегменты (1-2 секунды)
- WebRTC broadcast для "ближних" пиров
- Гибрид: первые пиры через direct WebRTC, остальные через торренты

**Клипы:**

```
Клип = ссылка на диапазон сегментов:
├── start: <timestamp>
├── end: <timestamp>
├── stream_id: <id оригинального стрима>
└── Проигрывается через тот же WebTorrent/IPFS
```

## 6. Wikipedia-подобное приложение (Wiki)

### Реализация на noet

```
Wiki = коллекция noet типа "wiki_page":
├── Каждая страница — CRDT (Yjs doc)
├── Коллективное редактирование в real-time
├── История версий: каждое изменение = новая версия в CRDT
├── Ссылки: wiki-ссылки на другие страницы (noet ID)
└── Тalk page: thread-noet для обсуждения
```

**Отличие от Wikipedia:**
- Нет центрального сервера
- Редактирование offline-first (CRDT merge)
- Каждое сообщество — своя wiki
- Форк страницы = форк noet

## 7. Блог-платформа (Medium-подобное)

### Реализация на noet

```
Блог = лента noet типа "article":
├── Rich text: Markdown/HTML в content
├── IPFS: медиа-файлы (изображения)
├── Комментарии: reply-noet
├── Подписки: GossipSub на автора
├── Теги: indexed для поиска
└── Репутация: награды за качественные статьи
```

## 8. DAO-приложение

### Реализация на noet

```
DAO = Сообщество + Treasury:
├── Предложения: proposal-noet
├── Голосование: vote-noet с весом от репутации/токенов
├── Treasury: Gnosis Safe (опционально) или native noet
├── Выплаты: подписанные транзакции после голосования
└── Прозрачность: все транзакции видны
```

## Ключевое преимущество

Все эти приложения **совместимы на уровне данных**:

- Пользователь из Discord-like приложения видит сообщения того же сообщества в Reddit-like
- Комментарий к видео (YouTube-like) — это reply-noet, видимый в любом клиенте
- Голосование в DAO — это vote-noet, проверяемый любым клиентом

noet — это не платформа с одним приложением. noet — это **интернет для сообществ**.
