# SCZ Philosophy: Философия Самоуправляемой Цифровой Зоны

## Метафизика: Что является первичным?

В иерархии SCZ первичным является **сообщество как сеть взаимосвязей между людьми**. Не платформа. Не протокол. Не данные. Не идентичность в вакууме. Именно отношения между людьми создают смысл всего остального.

### Иерархия первичности

```
Человек (субъект, обладающий ключами и правом выбора)
    ↓
Связь (отношения между людьми, доверие, репутация)
    ↓
Сообщество (сеть связей с общими правилами)
    ↓
Протокол (правила взаимодействия между узлами)
    ↓
Данные (информация, созданная и подписанная людьми)
    ↓
Узел (техническая единица инфраструктуры)
```

**Человек первичен.** Каждый человек владеет своей криптографической идентичностью. Никакой сервис не может отозвать, заблокировать или передать эту идентичность. Это цифровое продолжение личности, а не аккаунт, выданный корпорацией.

**Связь фундаментальна.** SCZ не существует без связей между людьми. Технологии обслуживают связи, а не заменяют их. Алгоритмы не определяют, с кем вы общаетесь. Вы сами определяете.

**Сообщество — единица суверенитета.** Отдельный человек имеет право выхода. Но коллективная жизнь требует правил. Сообщество — это минимальная единица, способная устанавливать и поддерживать правила.

**Протокол — инфраструктура, не власть.** Протоколы обеспечивают совместимость, но не навязывают правила. Правила определяет сообщество.

## 30 Принципов SCZ

### I. Принципы Идентичности (1-5)

**Принцип 1: Ключи — это личность**

Ваш публичный ключ — это вы. Не email, не телефон, не username, не домен. Если у вас есть приватный ключ, вы существуете в сети. Если вы потеряли ключ, вы потеряли идентичность. Резервное копирование — личная ответственность.

**Принцип 2: Идентичность неприкосновенна**

Никакой протокол, никакое сообщество, никакая технология не может лишить человека идентичности. Бан из сообщества — это потеря доступа к сообществу, не уничтожение личности.

**Принцип 3: Анонимность — это право**

Человек имеет право участвовать без раскрытия реальной личности. Репутация строится на действиях, а не на паспорте. Сообщества могут требовать идентификацию, но это их выбор, не требование системы.

**Принцип 4: Множественность личин**

Один человек может иметь множество идентичностей для разных контекстов. Семья, работа, хобби, политика — разные сферы могут требовать разных идентичностей. SCZ не требует единого профиля.

**Принцип 5: Восстановление возможно, но дорого**

Потеря ключей не должна быть катастрофой без возврата. Механизмы восстановления (социальное восстановление, мультиподпись, временные замки) должны существовать, но быть достаточно сложными, чтобы не подрывать безопасность.

### II. Принципы Сообщества (6-12)

**Принцип 6: Сообщество владеет собой**

Сообщество — это не группа пользователей продукта. Сообщество — это субъект, владеющий своими данными, правилами, инфраструктурой. Никакая внешняя сила не может навязать сообществу правила.

**Принцип 7: Право на выход**

Каждый участник имеет право покинуть сообщество с полным доступом к своим данным. Каждое меньшинство имеет право форкнуть сообщество с сохранением истории. Форк — не предательство, а эволюция.

**Принцип 8: Конституция выше кода**

Каждое сообщество имеет письменную конституцию — набор правил, принципов и процедур. Код реализует конституцию, но конституция определяет, что код должен делать. Если код противоречит конституции, конституция первична.

**Принцип 9: Правила прозрачны**

Все правила сообщества доступны всем участникам. Все решения о модерации публичны. Все голосования — открыты для проверки. Нет тайных правил. Нет теневых банов.

**Принцип 10: Модерация — не цензура**

Сообщество имеет право устанавливать правила поведения. Удаление контента по правилам, принятым сообществом, — это модерация. Удаление контента по внешнему давлению — цензура. Различие: кто принял решение.

**Принцип 11: Разнообразие культур**

SCZ не навязывает единые стандарты поведения. Одно сообщество может требовать вежливости, другое — поощрять провокации. Третье — быть посвящено науке, четвёртое — искусству. Единственное требование: правила должны быть записаны и применяться равномерно.

**Принцип 12: Сообщества конкурируют за участников**

Участники свободны выбирать сообщества. Сообщества конкурируют качеством правил, качеством дискуссий, качеством управления. Это рыночная дисциплина для цифровых обществ.

### III. Принципы Данных (13-17)

**Принцип 13: Данные принадлежат создателю**

Контент, созданный человеком, принадлежит этому человеку. Не платформе. Не сообществу. Не протоколу. Лицензия определяет создатель.

**Принцип 14: Данные самоудостоверяющиеся**

Каждый блок данных криптографически подписан. Подлинность проверяется без центрального сервера. Доказательство существования не требует доверия.

**Принцип 15: Данные переносимы**

Человек может забрать все свои данные и перейти в другое приложение, другое сообщество, другой клиент. Формат открыт. Нет vendor lock-in.

**Принцип 16: Данные устойчивы**

Данные не должны исчезать, если один узел выходит из строя. Репликация обеспечивает сохранность. Однако: право на забвение реализуется через шифрование, а не удаление.

**Принцип 17: Метаданные — тоже данные**

Кто с кем дружит, кто на что подписан, кто где комментировал — это чья-то информация. Метаданные должны быть защищены так же, как и контент. Граф связей принадлежит людям, не платформам.

### IV. Принципы Управления (18-23)

**Принцип 18: Разделение властей**

В каждом сообществе существуют различные роли: законодатели (принимают правила), исполнители (применяют правила), судьи (разрешают споры), аудиторы (проверяют соблюдение). Одна роль не может захватить всё.

**Принцип 19: Диктатура большинства — угроза**

Простое большинство голосов не может отнять права у меньшинства. Конституция защищает фундаментальные права. Изменение конституции требует квалифицированного большинства.

**Принцип 20: Право вето для заинтересованных**

Решения, затрагивающие конкретную группу, требуют согласия этой группы. Меньшинство не может навязать свою волю, но большинство не может разрушить меньшинство.

**Принцип 21: Постепенная децентрализация**

Новые сообщества могут начинаться с централизованного управления (основатель принимает решения). По мере роста управление передаётся сообществу. Это не баг — это фича. Ранняя стадия требует скорости, зрелая — демократии.

**Принцип 22: Управление наименьшей единицей**

Решения должны приниматься на самом низком возможном уровне. Глобальное голосование — крайняя мера. Локальные решения локальны. Федерализм, не унитарное государство.

**Принцип 23: Алгоритмы управления открыты**

Как считаются голоса? Как определяется репутация? Как распределяются ресурсы? Алгоритмы публичны, проверяемы, изменяемы через процедуру.

### V. Принципы Инфраструктуры (24-27)

**Принцип 24: Нет единой точки отказа**

Ни один узел, ни один разработчик, ни один фонд, ни одна юрисдикция не может остановить SCZ. Система работает, пока работает хотя бы два узла.

**Принцип 25: Полнота связи через избыточность**

Сеть остаётся связной, если у каждого узла достаточно соединений. Топология сети не зависит от одного маршрута. Пути обхода существуют всегда.

**Принцип 26: Слои независимы**

Идентичность не зависит от хранилища. Хранилище не зависит от транспорта. Транспорт не зависит от приложения. Каждый слой может быть заменён без разрушения остальных.

**Принцип 27: Простота протокола**

Протокол должен быть достаточно простым, чтобы его можно было реализовать одному разработчику за разумное время. Сложность — враг безопасности и долговечности.

### VI. Принципы Эволюции (28-30)

**Принцип 28: Форки — это здорово**

Форк не катастрофа, а механизм эволюции. Когда сообщество раскалывается, обе части сохраняют данные и историю. Конкуренция форков ведёт к лучшим решениям. Естественный отбор для цифровых обществ.

**Принцип 29: Технологии заменимы**

Конкретные библиотеки, языки программирования, форматы сериализации — всё это временно. Архитектурные принципы вечны. SCZ спроектирована так, чтобы пережить смену любой технологии.

**Принцип 30: Бессмертие через конституцию**

Сообщество переживает своих основателей, когда правила записаны, автоматизированы и переданы сообществу. Конституция — это геном сообщества. ДНК переживает организм.

## Принципы, которые нельзя нарушить

Ни при каких обстоятельствах:

1. **Никто не может забрать чью-либо идентичность.** Это включает основателей, разработчиков, государства, большинства голосов.

2. **Никто не может заблокировать форк.** Право на выход — фундаментально. Если сообщество раскалывается, обе стороны имеют право на данные.

3. **Протокол остаётся открытым.** Никакие патенты, никакие лицензии, никакие ограничения на использование протокола.

4. **Управление прозрачно.** Все правила, все решения, все алгоритмы — открыты для проверки.

5. **Код не может противоречить конституции.** Если код позволяет сделать то, что запрещено конституцией, это баг, а не фича.

## Философская основа

### Противопоставление

| Платформа | SCZ |
|---|---|
| Пользователи | Граждане |
| Аккаунты | Идентичности |
| Группы | Сообщества |
| Правила сервиса | Конституции |
| Бан | Исключение по правилам |
| Закрытие сервиса | Форк |
| Алгоритм ленты | Подписки + выбор |
| Монополия на правду | Множество культур |
| Одна лента для всех | Много лент, много правил |
| Платформа решает | Сообщество решает |
| Данные в облаке | Данные у людей |
| Зависимость от провайдера | Суверенитет |

### Этическая позиция

SCZ занимает позицию **либертарианского коммунитаризма**:

- **Либертарианский** — в отношении индивида: никакая сила не может принудить человека к участию
- **Коммунитарный** — в отношении сообществ: коллектив имеет право на самоопределение, включая право устанавливать правила входа
- **Федеративный** — в отношении масштаба: нет единого центра, есть сеть суверенных единиц

Это не ультимативная философия. Это прагматичный баланс между анархией (нет правил) и тоталитаризмом (одни правила для всех).

## Заключение

Философия SCZ — это не декларация намерений. Это архитектурные ограничения. Каждый принцип имеет техническую реализацию. Каждая ценность — конкретный механизм.

SCZ не обещает утопию. SCZ создаёт условия, при которых множество разных утопий (и антиутопий) могут сосуществовать, конкурировать, эволюционировать — без уничтожения друг друга и без поглощения корпорациями.
