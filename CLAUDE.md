# noet — правила проекта (read me before touching anything)

Проект: **noet** (noet) — цензуроустойчивая инфраструктура для
цифровых сообществ. Каноничные спецификации: `docs/01..10_*.md`. Главный клиент —
**кроссплатформенное приложение на Tauri v2** (Windows, Linux, Android) в `app/`.
Это файл-договор: нарушать пункты нельзя, даже если «и так сойдёт».

**Сначала прочти [`PRINCIPLES.md`](PRINCIPLES.md)** — абстрактный инженерный стандарт против
халтуры (поток пользователя, валидация, правила системы, фильтрация данных, UX на всю
высоту, живой UI без F5, чинить до корня). Применяется ко ВСЕМУ. Перед «готово» — чек-лист §10.

История: раньше клиентом было браузерное расширение (`poc/zone-portal/`, заморожено как
референс). Теперь клиент — нативное приложение. Причины перехода: §09_CLIENTS (магазины
расширений = точка отказа, нет мобильных, нет полноценного P2P/фона).

## Железные правила (нарушение = брак)

1. **Крипта — в Rust-бэкенде, не в вебвью.** Ключ secp256k1 живёт в `src-tauri` (файл в
   app-config-dir), подпись (BIP340 Schnorr) и генерация — только в Rust (`secp256k1`
   крейт). Фронтенд НИКОГДА не видит приватный ключ и не подписывает сам: он зовёт
   Tauri-команду `sign_event`. Бэкап ключа — явное действие пользователя (показать nsec
   один раз / скачать файл). Следствие: вебвью не нужен `crypto.subtle`, нет проблемы
   secure-context, которая мучила расширение.

2. **Проверяй в настоящем приложении, потом говори «готово».** GUI Tauri headless не
   прокликать, поэтому: Rust-логику (крипта, подпись, формат события) гонять юнит-тестами
   и `cargo test`/`cargo check`; фронтенд-логику — node-проверкой синтаксиса и, где можно,
   запуском `npm run tauri dev` на Linux. Если флоу (создание ключа, постинг, звонок) не
   прокликан вручную, он НЕ сделан. Android/Windows-сборку проверяет CI (GitHub Actions).

3. **Никаких сырых технических ошибок в UI.** Любой `await`/`invoke` оборачивать; ошибку
   Rust возвращать как понятный код, в UI показывать локализованный текст.
   `undefined is not an object` или `Err(...)` на экране = брак. Детали в консоль/лог, на
   экран — человеческое.

4. **i18n RU/EN на всех видимых поверхностях.** Тексты и ошибки через словарь
   (`app/src/i18n.js`, `t(key)`), а не хардкодом. Дефолт RU, переключатель есть.

5. **Одна личность, один индикатор.** Чип «ник+аватар» или «Гость» — один на все экраны
   (общий шелл/сайдбар), не плодить по вкладкам. Вход/создание/импорт/бэкап — на отдельном
   экране профиля, не формой в ленте.

6. **Детерминированные потоки.** Один ясный путь. Создал ключ → он сразу активен → дальше
   контент. Импорт — явная отдельная ветка для вернувшихся. Без «создай-или-импортируй-или-
   войди» в одной куче.

7. **Длинное тире `—` в КОНТЕНТЕ (тексты UI, строки словаря) не использовать.** Запятая,
   двоеточие, точка. (В коде/комментариях/доках допустимо.)

8. **Не использовать ник пользователя (`nyx`) как пример/плейсхолдер.** Нейтральные примеры.

9. **Думать наперёд:** пустые состояния, состояния ошибки, оффлайн (реле недоступны),
   мобильный экран (узкий, тач), что увидит человек, открывший приложение впервые.

10. **НЕ трогать живые данные при тестах.** Реальные ключи/события пользователя не трогать.
    Тестовые личности и события создавать в ОДНОРАЗОВОМ профиле (отдельный
    `app-config-dir` через env, или временная папка) и чистить только своё по pubkey.
    Публичные реле — общие, не флудить.

11. **Честные времянки — называть, не прятать.** Архитектура §04 целевая (libp2p, Hypercore,
    GossipSub, Yjs). Текущая Фаза-0 реализация прагматична и совпадает с уровнем расширения:
    сеть и обмен сообщениями — публичные **Nostr-реле** (wss), медиа — **IPFS-шлюзы**, noet —
    подписанные события. libp2p/DHT/Hypercore/CRDT — это РОАДМАП (§10, Фаза 0+), пока не
    реализованы. Не выдавать прагматичный слой за целевой: помечать TODO/времянку в коде и UI.

## Архитектура (ТЕКУЩАЯ: узел-бэкбон + зона в браузере)

ГЛАВНОЕ: приложение это НЕ «ещё одно приложение с вкладками». Это **узел (бэкбон)**,
играющий роль бывшего расширения: личность, связь, самообновление. **Всё остальное
(мессенджер, домены, сайты, игры, репутация) живёт в БРАУЗЕРЕ**, отдаётся узлом. Не
встраивать функционал зоны в нативное окно: окно тонкое, зона это веб.

- **Узел** = Tauri v2 (`app/src-tauri/`, Rust). Делает три вещи:
  1. **Личность** — ключ secp256k1 + подпись BIP340 Schnorr ТОЛЬКО в Rust (правило §1).
  2. **Связь** — локальный HTTP-шлюз (`gateway.rs`, `tiny_http`) на `127.0.0.1:8788+`,
     отдаёт встроенную зону (`include_dir!("../zone")`) и API:
     `/api/nostr/pubkey`, `/api/nostr/sign`, `/api/identity/{status,create,import,export,forget}`.
  3. **Самообновление** — `tauri-plugin-updater` (endpoints = GitHub Releases, pubkey в
     `tauri.conf.json`). Команды `check_update`/`install_update`. Для выпуска артефактов
     включить `createUpdaterArtifacts:true` + секреты `TAURI_SIGNING_PRIVATE_KEY[_PASSWORD]`.
- **Нативное окно** (`app/src/`, тонкий шелл): статус узла, адрес зоны, кнопка «Открыть noet
  в браузере» (`open_zone` → дефолтный браузер на адрес шлюза), проверка обновлений. Личность
  и приложения тут НЕ живут.
- **Зона** (`app/zone/`, ванильный JS, отдаётся шлюзом в браузер): мост `window.nostr`
  (подпись через `/api/nostr/*` узла) + `window.noet` (данные на публичных реле). Экраны:
  главная, мессенджер (лента+каналы), домены и сайты (занять имя, опубликовать сайт/игру
  под именем, открыть в iframe с мостом `noet-embed.js`), профиль + репутация (считается
  локально по событиям). Это «всё в браузере за счёт узла» вместо расширения.
- **Networking/Storage (Фаза-0, времянка §11)** = публичные Nostr-реле (wss) + IPFS-шлюзы.
  Имена и сайты публикуются как события (claim 31111, page 31002). Целевое — libp2p
  GossipSub + Hypercore (§04), миграция по роадмапу.
- **Realtime** = WebRTC P2P (host-star), сигналинг на реле; getUserMedia работает в браузере
  (зона на http://127.0.0.1 это secure context). Звонок/демонстрация экрана это зона.

## Структура

```
app/
  package.json            tauri-cli + скрипты (dev/build)
  src/                    НАТИВНОЕ ОКНО (тонкий шелл узла)
    index.html main.js styles.css logo.svg
  zone/                   ЗОНА (отдаётся шлюзом в браузер; ванильный JS)
    index.html            точка входа зоны
    noet.js                SPA: главная/мессенджер/домены-сайты/профиль+репутация
    noet-embed.js          мост window.nostr/noet для опубликованных сайтов и игр
    styles.css logo.svg
  src-tauri/              УЗЕЛ (Rust)
    Cargo.toml  tauri.conf.json  build.rs
    src/lib.rs            запуск, команды (gateway_url/open_zone/update), плагин updater
    src/gateway.rs        локальный HTTP-шлюз: зона + /api/identity + /api/nostr/sign
    src/identity.rs       secp256k1 keygen/sign
    capabilities/  icons/
.github/workflows/        CI: сборка Win/Linux/Android, релизы
docs/                     каноничные спецификации noet (01..10)
poc/zone-portal/          ЗАМОРОЖЕНО: старое браузерное расширение (референс)
```

## Команды

- Dev (Linux desktop): `cd app && npm install && npm run tauri dev`.
- Сборка desktop: `npm run tauri build` (→ `app/src-tauri/target/release/bundle/`).
- Android: `npm run tauri android init` затем `npm run tauri android build` (нужны Android
  SDK + NDK + JDK; в CI ставятся автоматически).
- Rust-проверка: `cd app/src-tauri && cargo check` / `cargo test`.
- Релизы и кроссплатформенные бинарники — через GitHub Actions (тег `v*` → Release с
  Windows `.msi`/`.exe`, Linux `.AppImage`/`.deb`, Android `.apk`).

## Что считается «уровнем расширения» (цель этой итерации)

Расширение умело: личность (ключ), профиль, ленту/реле, сообщества, вики (+удаление статей,
шапка везде), 1:1 и групповой звонок (камера + экран). Tauri-клиент должен повторить этот
набор на реле+IPFS-бэкенде, после чего по §10 двигаться к libp2p/Hypercore/CRDT.
