# noet — правила проекта (read me before touching anything)

Проект: зона **noet** (TLD `.nt`), цензуроустойчивое сообщество. Код PoC в `poc/zone-portal/`.
Это файл-договор: нарушать пункты нельзя, даже если «и так сойдёт».

## Железные правила (нарушение = брак)

1. **Secure context / крипта.** Зона работает по **http** на придуманных доменах (`id.nt`,
   `search.nt`, …). На таком origin **нет `window.crypto.subtle`** (WebCrypto доступен только
   на https или localhost/127.0.0.1). Поэтому:
   - НИКОГДА не использовать `crypto.subtle.*` в коде зоны.
   - Хеши (sha256 для id события Nostr) — только чистый JS (`web/sha256` / noble/hashes).
   - `crypto.getRandomValues` можно (работает и на insecure).
   - noble schnorr (`vendor/noble-secp256k1.js`) подписи делает без subtle — ок.

2. **Проверяй в настоящем браузере, потом говори «готово».** Не верить curl'у про UX.
   Гонять реальный сценарий (регистрация, постинг) через headless Chrome по CDP
   (`google-chrome-stable --remote-debugging-port`, навигация, скриншот). Если флоу не
   прокликан в браузере — он не сделан. Проверять и insecure-origin (через прокси), а не
   только 127.0.0.1.

3. **Никаких сырых технических ошибок в UI.** Любой `await` оборачивать; пользователю
   показывать понятный локализованный текст. `Cannot read properties of undefined` на
   экране = брак. Логировать детали в консоль, на экран — человеческое.

4. **i18n RU/EN на всех видимых поверхностях.** Тексты и ошибки через словарь
   (`web/i18n.js`, `window.t(key)`), а не хардкодом. Дефолт RU, переключатель есть.

5. **UX-разделение.** Регистрация/вход/профиль — на **отдельной странице `id.nt`**, не в
   строке поиска и не формой в виджете. На поиске и в реле — только чип «ник+аватар» или
   «Гость», по клику переход на `id.nt`. Виджет — навигация и показ личности, без форм входа.

6. **Детерминированные потоки.** Один ясный путь, без «создай-или-импортируй-или-войди» в
   одной куче. Создал ключ → он сразу активен → дальше только хэндл+инвайт. Импорт — явная
   отдельная ветка для вернувшихся.

7. **Длинное тире `—` в контенте не использовать.** Запятая/двоеточие/точка.

8. **Не использовать ник пользователя (`nyx`) как пример/плейсхолдер.** Нейтральные примеры.

9. **Думать наперёд:** secure-context, кросс-ориджин, пустые состояния, состояния ошибки,
   мобайл, что увидит человек, который видит это впервые.

## Архитектура (origins и ключ)

- Доступ по имени: расширение ставит PAC `*.nt → PROXY 127.0.0.1:8090`. Портал
  (`server.mjs`) резолвит имя → CID → IPFS и отдаёт статику зоны.
- **Ключ личности (Nostr) живёт только в origin `id.nt`** (localStorage). Это:
  - видимая **страница аккаунта** (регистрация/вход/профиль/бэкап/мои имена) — всё in-origin;
  - одновременно **мост**: когда `id.nt` встроен скрытым iframe в другую страницу зоны,
    отвечает на `postMessage` — `whoami` (чтение, любому origin) и `signEvent`
    (подпись, только доверенным `search/relay/profile/id`).
- `relay.nt` — настоящее Nostr-реле поверх WebSocket (`ws.mjs`, NIP-01). Профиль = kind 0,
  сообщения = kind 1. Реле постит kind1, подписывая через мост `id.nt`.
- `search.nt` — поиск, только чип личности. Виджет (`web/widget.js`, Shadow DOM) — на всех
  страницах: показывает личность, навигация, прячется (Alt+N), без форм.

## Команды

- Портал/IPFS — systemd user units `scz-portal`, `scz-ipfs` (порт 8090, gw 8080).
  `systemctl --user restart scz-portal` после правок server.mjs.
- Статика `web/*` отдаётся с диска свежей, перезапуск не нужен; расширение трогаем только
  при смене PAC/иконки.
- Инвайт (основатель): `curl -X POST -H "x-admin-token: <ADMIN>" http://127.0.0.1:8090/api/invite`.
- Скриншот флоу: CDP через `google-chrome-stable --headless=new --remote-debugging-port=NNNN`.

## Честные времянки (назвать, не прятать)

Привратник = основатель (выдаёт инвайты). Реестр имён = сервер-судья. Ключ в localStorage,
бэкап обязателен, паскей-восстановление — следующий шаг. Всё это §2.1/§7.2 концепции.
