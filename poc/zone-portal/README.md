# noet

**Свободный интернет прямо в браузере: сайты, которые нельзя заблокировать, удалить или отобрать.**

noet это дополнение для браузера, которое даёт тебе собственные домены и сайты. Они
привязаны к твоему ключу и принадлежат тебе: их не арендуют у регистратора и не выдаёт
компания, которая может их отозвать. Центрального сервера нет, каждая страница живёт в сети
и подписана тобой, поэтому её нельзя заблокировать, удалить или отобрать. Личность это твой
ключ: ни почты, ни паролей.

Страница установки: **https://noet-scz.github.io/noet/**

---

## Установка

Нужен компьютер. На телефонах пока нельзя: мобильные браузеры не дают ставить такие
дополнения.

### Vivaldi, Chrome, Brave, Edge

1. Скачай **[noet-chrome.zip](https://noet-scz.github.io/noet/dist/noet-chrome.zip)**.
2. Распакуй архив в любую папку. Не удаляй её потом: браузер читает дополнение прямо из неё.
3. Открой страницу расширений: `chrome://extensions` (в Vivaldi `vivaldi://extensions`,
   в Edge `edge://extensions`).
4. Включи **Режим разработчика** (переключатель справа вверху).
5. Нажми **Загрузить распакованное** и выбери распакованную папку.
6. Набери в адресной строке `noet.nt`. Готово.

### Firefox

1. Скачай **[noet-firefox.xpi](https://noet-scz.github.io/noet/dist/noet-firefox.xpi)**.
2. Открой `about:debugging#/runtime/this-firefox`.
3. Нажми **Загрузить временное дополнение** и выбери скачанный `.xpi`.
4. Набери `noet.nt`.

В Firefox дополнение пока ставится временно и слетает при перезапуске браузера. Постоянная
версия будет, когда дополнение подпишут. Если хочешь, чтобы работало всегда, ставь в Chrome,
Vivaldi, Brave или Edge.

## Первые шаги

1. **Создай личность.** Нажми иконку noet рядом с адресной строкой и выбери «Создать
   личность». Дополнение само скачает файл `noet-ключ.txt` с твоим ключом.
2. **Сохрани этот файл.** Это твой единственный вход в сеть: ни пароля, ни восстановления по
   почте нет. Потеряешь ключ, потеряешь личность и все свои страницы. Положи файл в надёжное
   место, например в менеджер паролей.
3. **Осмотрись.** Набирай в адресной строке:
   - `noet.nt` поиск по сети
   - `id.nt` твой профиль
   - `people.nt` участники и репутация
   - `relay.nt` общая лента
   - `dev.nt` для тех, кто хочет собирать своё
4. **Сделай свою страницу.** На `id.nt` задай имя и аватар. Потом нажми «Мои домены», займи
   себе имя (например `alex`) и отредактируй его страницу. Адрес `alex.me` станет твоим. Под
   своим именем можно делать и поддомены: `blog.alex.me`, `shop.alex.me`.

## Что это и как устроено

Нет платформы и центрального сервера, который кто-то может выключить или захватить.

- **Личность** это пара ключей. Приватный ключ лежит в дополнении, у тебя. Им подписывается
  всё, что ты делаешь.
- **Содержимое страниц** лежит в IPFS и адресуется по содержимому: одна и та же страница
  доступна с любого узла сети.
- **Имена** (`имя.me`, `имя.nt`) закрепляются за ключом подписанной заявкой на публичных
  Nostr-реле. Очередь на спорное имя решают метки времени Bitcoin (OpenTimestamps), без
  чьего-либо разрешения.
- **Репутация** считается у тебя в браузере из подписанных событий. Она даёт ёмкость
  (например, сколько имён можно держать), а не власть над другими.

Заблокировать или удалить страницу нечем: нет единой точки, через которую это делается.
Можно скачать полную копию состояния и поднять свой экземпляр сети.

## Собрать своё

В каждой странице noet доступен `window.noet` с пятью примитивами (личность, пространство,
запись, связь, политика). Из них собираются форумы, блоги, доски, вики, игры, не меняя ядро.
Готовые шаблоны лежат в [`sites/`](sites/).

Целый собранный проект (например результат `npm run build`) заливается под твоё имя одной
командой, см. вкладку **«Разработчикам»** (`dev.nt`) в дополнении или
[`tools/noet-publish.mjs`](tools/noet-publish.mjs).

## Разработка

Сборка дополнения: `node build-ext.mjs` создаёт `dist/noet-chrome.zip` и
`dist/noet-firefox.xpi`. Во время разработки удобно грузить папку `ext/` напрямую через
«Загрузить распакованное».

| Компонент | Что |
|---|---|
| `ext/` | дополнение: перехват имён, sandbox-рендер, `window.nostr`/`window.noet`, ключ |
| `dist/` | конфиг, зеркало имён, сборки расширения (GitHub Pages) |
| `tools/` | `noet-publish.mjs`: залить собранный проект под своё имя |
| `sites/` | примеры: гостевая, форум, блог, доска, вики, дашборд, кликер |
| `docs/` | архитектура, философия, федерация, роадмап |
| `server.mjs`, `auth.mjs`, `gov.mjs`, `ws.mjs` | исторический реестр; для работы сети больше не обязателен (имена и данные живут на публичных реле и в IPFS) |
