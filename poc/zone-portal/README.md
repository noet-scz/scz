# Zone Portal — зона .scz: поиск + доступ по имени в браузере

Доступ к сайтам зоны **по имени, в самом браузере**, без виджета:
`http://search.scz/`, `http://manifest.scz/`, `http://practices.scz/`.

Имена `*.scz` резолвит зона (не DNS): PAC отправляет такие хосты на прокси-резолвер,
тот резолвит имя → CID → отдаёт страницу из IPFS.

## ⚠️ Про DPI (важно, проверено вживую)

На сети с DPI (как у тебя, с Zapret) прямой http к VPS **режется по cleartext-Host**:
запрос с `Host: manifest.scz` к `144.31.25.136:8090` отдаёт пустой ответ, а тот же
сервер с самого VPS работает. Это ровно §9.2 концепции: труба по IP открыта, но DPI
смотрит на форму трафика (заголовок Host в открытом HTTP).

**Обход без домена и сертификата — туннелировать прокси-хоп через SSH.** Браузер ходит
на `127.0.0.1`, SSH несёт всё шифрованным до VPS, DPI слепнет.

## Как включить (3 шага)

### 1. Поднять SSH-туннель (держать открытым)
```fish
ssh -N -L 8090:127.0.0.1:8090 root@144.31.25.136
# или: ./tunnel.sh   (пароль — из 09_auth.md, server #2)
```
Совет: добавь свой ssh-ключ на сервер, чтобы не вводить пароль каждый раз.

### 2. Включить резолвинг .scz (любой из вариантов)

**A. Расширение (Chromium: Chrome/Brave/Edge)**
- `chrome://extensions` → включить «Режим разработчика» → «Загрузить распакованное» →
  выбрать папку `extension/`.
- Иконка покажет `ON`. Клик — вкл/выкл. (Расширение уже указывает на `127.0.0.1:8090`.)

**B. Вручную через PAC (Firefox и любые)**
- Настройки прокси → «URL автоматической настройки прокси (PAC)»:
  `http://127.0.0.1:8090/scz.pac`
- PAC сам вернёт `PROXY 127.0.0.1:8090` (берётся с того адреса, по которому скачан).

### 3. Открывать по имени
В адресной строке (именно `http://`, зона по http):
```
http://search.scz/        ← поисковик зоны
http://manifest.scz/      ← сайт
http://practices.scz/     ← сайт
```
Поиск и переходы между сайтами (`http://...scz/`) — обычная навигация браузера. Виджета нет.

## Как устроено

```
браузер → PROXY 127.0.0.1:8090 ─[SSH-туннель, шифр.]→ VPS:8090  zone-portal (Node)
   *.scz                                                  ├─ search.scz → поисковик
   остальное → DIRECT (обычный интернет)                  ├─ <имя>.scz → сайт из IPFS
                                                          ├─ /scz.pac  → PAC
                                                          └─ /api/search, /api/names
                                                                  │ docker-сеть
                                                                  ▼
                                                          scz-ipfs gateway → IPFS по CID
```

Поисковик краулит зарегистрированные `*.scz` сайты из IPFS, строит индекс (раз в 5 мин),
ищет по вхождениям (тело + бонус за `<title>`). Резолвер тянет контент через локальный
IPFS-узел, без публичных шлюзов.

## Участие: личность и вход (гейт только на участие)

Читать и искать может любой. Чтобы **участвовать** (занять имя, позже — постить) —
нужен вход. Личность = **Nostr-ключ** (§4.1), вход — подписью события (NIP-42-стиль),
сервер проверяет schnorr-подпись. Регистрация нового ключа требует **инвайта** (раздаёт
основатель — времянка-привратник, §2.1) и уникального хэндла.

Страница: **`http://search.scz/id`** (или напрямую `http://127.0.0.1:8090/id`).
- Есть расширение Nostr (NIP-07, напр. Alby/nos2x) → вход из него, ключ не покидает расширение.
- Нет → ключ создаётся в браузере; **обязательно скачать бэкап** (потерял ключ — потерял
  личность, §7.2; паскей-восстановление — следующий шаг).

Основатель выпускает инвайты (нужен `ADMIN_TOKEN` из systemd drop-in):
```bash
curl -X POST -H "x-admin-token: <TOKEN>" http://127.0.0.1:8090/api/invite
```

Эндпоинты: `GET /api/auth/challenge` · `POST /api/auth/login {event,handle,invite}` ·
`GET /api/me` · `POST /api/claim {name,cid}` (нужен вход) · `POST /api/invite` (admin).
Крипта — вендорённый `@noble/curves` (schnorr/BIP340), `vendor/noble-secp256k1.js`.

## Честные оговорки
- **DPI** режет открытый http по Host → нужен SSH-туннель (или TLS-домен — это следующий
  шаг устойчивости, тогда туннель не нужен).
- Реестр имён — **времянка** (судья очерёдности = сервер). Честный вариант — L2-контракт.
- Зона по **http** (без TLS) — внутри туннеля это не страшно; наружу за TLS-доменом — потом.
- Публичная регистрация имён закрыта (нужен `ADMIN_TOKEN`); IPFS RPC API наружу не выставлен.
