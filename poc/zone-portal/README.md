# Zone Portal — поисковик + резолвер зоны + расширение

Поисковая система **внутри зоны СЦЗ** + браузерное «расширение» (Tampermonkey),
чтобы зона работала в обычном браузере. Развёрнуто на VPS.

## 🔗 Как проверить (живой стенд)

**Адрес портала:** http://144.31.25.136:8090

### Вариант A — без расширения, прямо в браузере
Открой `http://144.31.25.136:8090/` — поиск по зоне + каталог сайтов. Ищи, например:
`метакогниция`, `право форка`, `IPFS Nostr`, `цензуроустойчивость`. Клик по результату
рендерит сайт зоны (контент берётся из IPFS по хешу, не с сервера).

### Вариант B — расширение (то, что просил проверить)
1. Поставь **Tampermonkey** в браузер.
2. Открой `http://144.31.25.136:8090/scz-zone.user.js` → Tampermonkey предложит установить.
   (При первом запросе разреши доступ к домену `144.31.25.136`.)
3. На **любой** странице появится плавающая кнопка **⬡** (справа внизу) или жми **Alt+Z**.
4. В оверлее:
   - **адресная строка зоны** — введи `манифест.зона` или `практики.зона` → откроется сайт;
   - **поиск** — ищи по зоне, клик по результату открывает сайт;
   - внутри сайтов внутризонные ссылки (`*.зона`) кликабельны и ведут по зоне.
5. Бонус: на обычных страницах ссылки, ведущие на `*.зона`, перехватываются и открываются в зоне.

Сайты-заглушки: **манифест.зона**, **практики.зона**.

## Как устроено

```
браузер ──(Tampermonkey, GM_xmlhttpRequest, http без mixed-content)──┐
                                                                      ▼
   http://144.31.25.136:8090   zone-portal (Node, без зависимостей)
        ├─ /                    поиск + каталог (работает и без расширения)
        ├─ /scz-zone.user.js    юзерскрипт (адрес VPS подставляется на лету)
        ├─ /api/search?q=       поиск по индексу
        ├─ /api/names           каталог
        ├─ /raw/<имя>           index.html сайта (для рендера в srcdoc)
        └─ /r/<имя>             серверная обёртка
                    │
                    ▼  (docker-сеть scz-net)
        scz-ipfs gateway :8080 ──→ контент по CID из IPFS
```

- **Поисковик** краулит зарегистрированные `*.зона` сайты (тянет index.html из IPFS,
  снимает теги, строит индекс), переиндексация каждые 5 мин. Поиск — по вхождениям
  термина (тело + бонус за совпадение в `<title>`), со сниппетами.
- **Резолвер** отдаёт сайт по имени из IPFS-узла, **без публичных шлюзов**.
- **Расширение** ходит на портал через `GM_xmlhttpRequest` (обходит mixed-content, т.к.
  портал по http), рендерит зону в изолированном `srcdoc`-iframe.

## Честные оговорки

- Реестр имён — **времянка** (судья очерёдности = сервер/мультисиг). То же ограничение,
  что и в `../registry/` — честный вариант это L2-контракт (Фаза 2).
- Портал по **http на голом IP** (без TLS) — для PoC ок; в норме за TLS-доменом.
- Публичная регистрация имён **закрыта** (нужен `ADMIN_TOKEN`) — из интернета в реестр
  не пишут. Имена сею при деплое.
- IPFS RPC API наружу не выставлен; портал ходит в IPFS только через gateway по docker-сети.
