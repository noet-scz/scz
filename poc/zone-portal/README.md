# noet — зона .nt: поиск, личность, зала

Самоуправляемая цифровая зона с доступом **по имени прямо в браузере**:
`http://search.nt/`, `http://relay.nt/`, `http://manifest.nt/`, `http://practices.nt/`.

Имена `*.nt` резолвит зона (не DNS): расширение ставит PAC, по которому такие хосты
идут на локальный резолвер (порт 8090), тот резолвит имя → CID → отдаёт страницу из IPFS.

## Что есть

- **search.nt** — минималистичный поиск по зоне. То же поле открывает имя (`manifest.nt`).
- **relay.nt** — общая зала: настоящий **Nostr** (подписанные события поверх WebSocket-реле),
  ники, аватары в кружках, живая лента. Читать может каждый, писать — после входа.
- **id.nt** — служебный origin личности. Ключ хранится только здесь, в изоляции; другие
  сайты зоны его не видят и не могут подписать что-то молча.
- **Виджет** — на каждой странице зоны: плавающая кнопка в углу. Перетаскивается, прячется
  в стрелку у края (или хоткеем **Alt+N**), ведёт в профиль, настройки, поиск и залу,
  показывает твой аватар.
- **Личность = ключ** (Nostr). Вход подписью события; регистрация нового ключа требует
  инвайта (раздаёт основатель — времянка-привратник) и уникального хэндла.

## Установить / обновить расширение

Распакованное расширение Chromium (Vivaldi/Chrome/Brave/Edge), папка `extension/`:

1. `chrome://extensions` (в Vivaldi: `vivaldi://extensions`) → включить **Режим разработчика**.
2. **Загрузить распакованное** → выбрать папку `extension/`. Иконка покажет `ON`.
3. **Обновление:** после изменений в `extension/` нажми кружок-стрелку «Обновить» на карточке
   расширения (или сними и поставь заново). Распакованные расширения сами не обновляются;
   автообновление возможно только из магазина или со своего `update_url` (следующий шаг).

> Сменили TLD с `.scz` на `.nt` — старое расширение надо **обновить/переустановить**,
> иначе PAC будет слать на старый домен.

Альтернатива без расширения (Firefox и любые): настройки прокси → «URL автонастройки (PAC)»
= `http://127.0.0.1:8090/nt.pac`.

## Как устроено

```
браузер → PROXY 127.0.0.1:8090  zone-portal (Node, server.mjs)
  *.nt                              ├─ search.nt / relay.nt / id.nt → web/*.html
  остальное → DIRECT                ├─ <имя>.nt → сайт из IPFS (+ инжект /widget.js)
                                    ├─ /api/* → поиск, имена, участники, вход, claim
                                    ├─ WebSocket /relay → Nostr-реле (ws.mjs)
                                    └─ /nt.pac
                                          │ docker/local
                                          ▼
                                    IPFS gateway → контент по CID
```

Личность работает на всех `.nt` так: виджет/страница держат скрытый iframe на `id.nt` и
общаются с ним через `postMessage`. Ключ и подпись — только внутри `id.nt`. Подпись/логин
разрешены лишь доверенным origin'ам (`search/relay/profile/id`), чтение личности (аватар,
ник) — любому сайту зоны.

## Файлы

- `server.mjs` — портал: реестр, резолвер, поиск, личность, статика, прокси `*.nt`.
- `ws.mjs` — Nostr-реле поверх WebSocket (RFC6455 вручную, без зависимостей).
- `auth.mjs` — вход подписью, инвайты, хэндлы, проверка Nostr-событий.
- `web/` — `search.html`, `relay.html`, `bridge.html`, `widget.js`, `bridge.js`, `app.css`, `logo.svg`.
- `extension/` — расширение (PAC `*.nt → PROXY 127.0.0.1:8090`) + иконки.
- `sites/manifest`, `sites/praktiki` — сайты-заглушки (в IPFS).
- `vendor/noble-secp256k1.js` — schnorr/BIP340 (аудированный `@noble/curves`).

## Честные оговорки

- **Привратник = основатель** (выдаёт инвайты) — спрятанный суверен назван, времянка.
  Честный вариант (репутация/L2) — позже.
- **Реестр имён** — времянка (судья очерёдности = сервер). Честный вариант — L2-контракт.
- **Ключ в браузере**: сгенерённый ключ лежит в localStorage origin `id.nt`. Бэкап
  обязателен. Паскей-восстановление — следующий шаг.
- Зона по **http** (локально это не страшно). Наружу за TLS-доменом — отдельный шаг;
  на DPI-сети открытый http по доменному Host режется (§9.2), для удалённого доступа
  нужен туннель или TLS.
- Реле хранит события в памяти + дамп `relay-events.json` (последние тысячи). Федерация
  с внешними Nostr-реле возможна, но по умолчанию зона варится в своём.

## API

`GET /api/search?q=` · `GET /api/names` · `GET /api/members` · `GET /api/resolve/:name` ·
`GET /api/auth/challenge` · `POST /api/auth/login {event,handle,invite}` · `GET /api/me` ·
`POST /api/claim {name,cid}` (нужен вход) · `POST /api/invite` (admin) ·
WebSocket `/relay` (NIP-01: `REQ`/`EVENT`/`CLOSE`).

Выпустить инвайт (основатель):
```bash
curl -X POST -H "x-admin-token: <ADMIN_TOKEN>" http://127.0.0.1:8090/api/invite
```
