Для разработчиков

Всё, что нужно, чтобы построить своё на noet: страницы, игры, доски, дашборды. Сервер для этого не нужен.

ПоискЛюдиАккаунт

Конструктор: пять примитивов

На каждой странице, открытой через noet, доступен window.noet — конструктор из пяти примитивов. Из них собирается что угодно (форум, блог, доска, вики, маркет), не меняя ядро. Данные это подписанные события: они принадлежат автору, живут в общей сети и переживают любой отдельный сервер. Версия SDK: noet.version.

Identity — кто действует

const me = await noet.identity.me();          // { pubkey }
const profile = await noet.identity.get(pubkey); // { name, picture, about }

Space — пространство (любой тип)

// type это строка, ядро её не толкует: feed / forum / board / wiki / market / page …
const space = await noet.space.create({ type: "forum", title: "Мой форум" });
const s = await noet.space.get(space.coord);
const mine = await noet.space.list({ owner: me.pubkey });

Artifact — содержимое

// мелкое (текст/json) = событие; крупное (файл) = CID в IPFS, ссылка в событии
await noet.artifact.publish({ space, content: "первый пост" });
await noet.artifact.publish({ key: "score", content: "1240", topic: "my-game" }); // изменяемое
const feed = await noet.artifact.query({ space });        // лента пространства
const board = await noet.artifact.query({ topic: "my-game" });

Relation — связи

await noet.relation.add({ type: "reply", to: postId, toAuthor: pk, content: "ответ" });
await noet.relation.add({ type: "react", to: postId, content: "🔥" });
const replies = await noet.relation.query({ to: postId });

Policy — правила

await noet.policy.set(space, { post: { minRep: 3 }, mods: [pk] });
const rules = await noet.policy.get(space);
const ok = await noet.policy.can({ scope: space, action: "post", pubkey: me.pubkey }); // подсказка UI

Низкий уровень (если нужно своё)

await noet.publish({ kind: 1, tags: [], content: "привет" }); // подписать + разослать в реле
const events = await noet.query({ kinds: [1], authors: [pk] });

Готовые шаблоны (форк и деплой)

Разные продукты на ОДНИХ примитивах — ядро не меняется, меняется только разметка снаружи. Открой исходник, скачай папку, задеплой через аккаунт под своим поддоменом (напр. forum.твойтег.me), при желании задай тему в noet.env.json ({"APP":"моё-имя"}). Это и есть смысл noet: ты строишь своё.

ШаблонПримитивыИсходник
Гостевая книгаSpace feed + Artifact + Relationоткрыть
Форумтемы (Artifact) + ответы (Relation reply)открыть
Блогстатьи (Artifact, лонгрид 30023)открыть
Объявления / маркетлоты (Artifact) + отзывы (Relation)открыть
Викистраницы по ключу (Artifact 30078, версии) + [[ссылки]]открыть
Дашборд / Играсобытия + аналитика (Artifact)дашборд · кликер

Поменяй разметку форума — получится Q&A или баг-трекер. Добавь к блогу комментарии (Relation) — выйдет медиа. Ни одна правка не трогает ядро noet.

window.nostr — подпись (NIP-07)

Стандартный интерфейс подписи. Ключ живёт в расширении и не попадает на страницу.

const pk = await window.nostr.getPublicKey();
const signed = await window.nostr.signEvent({ kind: 1, tags: [], content: "привет" });

Аналитика в одну строку

Подключи снипет, и события посещений попадут в твой дашборд (кнопка «Создать дашборд» в аккаунте).

<script src="https://noet-scz.github.io/noet/dist/noet-analytics.js" data-app="ТВОЙ_ТЕГ" data-auto></script>
// или вручную: noetTrack("click", { button: "play" })

HTTP API реестра

ЧтоЗапрос
Все именаGET /api/names
Имя → CIDGET /api/resolve/<имя>
ПоискGET /api/search?q=
Участники + репутацияGET /api/people
Доступы и правилаGET /api/gov
Полная копия (форк)GET /api/export
Кто яGET /api/me + Authorization: Bearer <токен>
Опубликовать страницуPOST /api/publish { title, body } или { mode:"html", body }
Задеплоить проектPOST /api/publish-dir { sub, files:[{path, data(base64)}] }
ПоручитьсяPOST /api/vouch { event: kind 8001, p=за кого } (8002 = отозвать)
Дать доступPOST /api/delegate { event: kind 8010, p=кому, scope, until?, redelegate? }
Отозвать доступPOST /api/revoke { event: kind 8011, e=id доступа }
Создать инвайтPOST /api/invite { event: kind 8020 } (нужен доступ «инвайты»)
Обновить страницу зоныPOST /api/publish-zone { name, body } (нужен доступ к имени)

События управления (8001–8020) это обычные подписанные Nostr-события. Реестр проверяет подпись и право, всё хранится открыто и попадает в /api/export.
