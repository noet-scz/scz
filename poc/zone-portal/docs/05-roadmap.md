# 05 — Роадмап

Переход от «реестр хранит всё» к «конструктор из примитивов». Каждая фаза: цель, стек, todo,
выходной тест (как у концепции §11 — проверяемый, не «сделано на глаз»), анти-платформенная
проверка. Оценка в **единицах работы**, не в календаре.

Порядок не жёсткий, но P1→P2 это фундамент федерации; без них «забрать с собой» формально.

---

## P0 — Сделано

Личность (Nostr-ключ в расширении, NIP-07), имена в реестре, страницы и проекты в IPFS,
sandbox-рендер с JS, реле-чат, репутация-ёмкость + делегации доступа (gov.mjs), страницы
people/dev, право форка как `/api/export`. Владелец экземпляра назван.

---

## P1 — SDK примитивов (поверхность конструктора) — СДЕЛАНО

**Цель:** превратить window.noet (me/publish/query) в полный SDK пяти примитивов, чтобы
снаружи собирался любой продукт без изменения ядра.

**Стек:** `ext/noet-primitives.js` — единый источник правды, наращивает window.noet поверх
ядра; работает И в content-script (обычные страницы), И в sandbox-рендере (view.js передаёт
исходник в render.html). Nostr kinds: 0/1/3/7/30023/1063/30078/31000–31002. Публичные wss-реле.

## P5 — Имена без сервера (OpenTimestamps) — СДЕЛАНО (ядро)

**Цель:** убрать последнюю серверную власть — судью очерёдности имён. Имя-заявка с
OTS-якорем; победитель — самая ранняя; считает клиент.

**Стек:** свой компактный OTS (без тяжёлой библиотеки): штамп = POST дайджеста в
calendar-серверы (CORS `*`, работает с http-страницы); проверка = парсер сериализованного
Timestamp (sha256/ripemd160/append/prepend) → Bitcoin-аттестация → сверка merkle с реальным
блоком через blockstream/mempool. `ext/noet-names.js` (портативно node+браузер).

**Todo:**
- [x] Заявка имени = подписанное событие 31111; OTS-proof в ОТДЕЛЬНОМ событии 31112 (e=id заявки), чтобы не ломать подпись и чтобы апгрейдил кто угодно
- [x] Штамп id заявки в OTS-календарь браузером пользователя (account.js, фоном при публикации)
- [x] Клиентский резолвер `resolveName`: 1 претендент → он; коллизия → самая ранняя проанкоренная в Bitcoin, провизорные позади, ничья по id
- [x] Полная trustless-проверка: парсер OTS + сверка merkle с реальным блоком Bitcoin (проверено на фикстуре hello-world → блок 358391)
- [x] view.js: бессерверный путь — если имени нет в индексе-зеркале, резолв по заявкам с публичных реле
- [x] Честная страница «Имена без сервера / чем жертвуешь, что получаешь» на manifest.nt
- [ ] Снапшоты лога заявок в IPFS для durability; автоапгрейд pending→bitcoin фоном

**Выходной тест:** ✓ live e2e в node: подписал заявку → штамп в OTS-календарь → опубликовал
заявку+proof на публичные реле → НЕЗАВИСИМЫЙ читатель резолвит имя→владельца БЕЗ сервера.
Bitcoin-верификация доказана на готовой фикстуре. Порядок коллизий покрыт юнит-тестами.

**Анти-платформа:** ✓ убирает спрятанного суверена корня имён в нейтральные часы (Bitcoin).
