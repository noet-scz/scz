# 04 — Архитектура

## Слои (цель)

```
СЛОЙ 4 — ДОСТУП
  Расширение браузера: универсальный рендерер + подписант.
  Перехватывает имена → строит ВИД из событий+IPFS. Инжектит window.noet / window.nostr.

СЛОЙ 3 — КОНСТРУКТОР
  window.noet SDK: identity / space / artifact / relation / policy.
  Шаблоны-продукты (форум, блог, вики…) в IPFS, не в ядре.

СЛОЙ 2 — ПРИМИТИВЫ
  Подписанные события (Nostr) на ПУБЛИЧНЫХ реле + кастомные kind noet.
  Политика/репутация = детерминированные функции над событиями.

СЛОЙ 1 — ДАННЫЕ И ИМЕНА
  Контент: IPFS (CID). Социальное/состояние: реле.
  Имена: оракул name→координата (сейчас сервер-судья → цель OpenTimestamps).
```

Принцип: **чем выше слой, тем больше его можно заменить, не трогая нижние.** Продукт
(слой 3) полностью снаружи. Ядро (слой 2) не знает про продукты.

## Реестр-как-оракул (главный редизайн)

Сейчас реестр делает четыре работы: имена, личность/сессии, реле, указатели контента. Это
силос. Цель — оставить ему **одну** работу и одну вспомогательную:

- **Оракул имён:** `name → координата` (ключ / адресуемая координата Space / CID) с
  уникальностью. Это единственное, чему нужен «судья очерёдности» (§4.3).
- **Сидер (вспомогательно):** пиннит CID участников и анонсирует в DHT, чтобы контент жил
  при офлайн-авторе. Не источник истины — кэш-помощник.

Что **уезжает из реестра** в события на публичных реле:

| Сейчас в реестре | Куда переезжает |
|---|---|
| Личность, сессии (`members.json`, auth) | События Identity (kind 0/3/22242/80xx), ключ в расширении |
| Лента реле (`relay-events.json`) | Публичные реле + опционально своё реле как одно из многих |
| Указатель `name→CID`, `raw` контента | Artifact-событие автора (31002 → CID); реестр держит только `name→координата` |
| Поручительства/доступы (`gov.json`) | События Policy (80xx) на реле; функция репутации считает их клиентски |

После переезда **форк перестаёт расходиться** (см. [03-federation.md](03-federation.md)):
экземпляр — это вид, а не хранилище.

## Расширение-как-рендерер

Расширение уже: перехватывает имена, рендерит контент из IPFS в sandbox-странице (где
работает JS), инжектит window.nostr (подпись ключом) и window.noet (данные через реле).

Доращиваем до универсального клиента-конструктора:

- **window.noet расширяется** с `me/publish/query` до полного SDK примитивов (см.
  [02-primitives.md](02-primitives.md)).
- **Рендер по типу Space:** имя ведёт к Space; расширение выбирает шаблон по `type`
  (page/feed/forum/…). Шаблоны — обычные сайты в IPFS, не код расширения. Расширение лишь
  даёт им примитивы и подпись.
- **Имя резолвится в координату, не только в CID:** `alice.me` → Space-координата →
  шаблон + поток артефактов.

## Реле: своё как одно из многих

Своё ws-реле (ws.mjs) остаётся, но **перестаёт быть домом данных.** Данные живут на пуле
ПУБЛИЧНЫХ реле (переносимость), своё реле — это комьюнити-реле и ускоритель. Список реле
сообщества публикуется (часть «корня», как app_hosts).

## Стек: что берём, что пересматриваем

| Компонент | Решение | Почему |
|---|---|---|
| Личность, события, связи, политика | **Nostr** | Готовая сеть, ключи без аккаунтов, тупые взаимозаменяемые реле (§4.1) |
| Крупный контент | **IPFS (kubo)** | Адресация по содержимому, CID одинаков везде (§4.2) |
| Клиент/рендер/подпись | **Расширение MV3** (Chrome осн., Firefox статика) | Перехват имён + sandbox-рендер JS + NIP-07 |
| Адресуемые/изменяемые объекты | **NIP-33 (30000–39999)** | Страница/Space/Policy обновляются переизданием, имя ведёт к свежему |
| Данные приложений | **NIP-78 (30078)** | Произвольные артефакты без новых kind ядра |
| Имена (уникальность) | сейчас сервер-судья → **OpenTimestamps** | Гаслесс нейтральные часы (Bitcoin), серверу можно умереть |
| Хранение «своих» | пиннинг участников + бэкап-сервис как сетка | Контент жив, пока пиннят (§4.2) |
| Приватность | **NIP-44** (шифр), **NIP-87** (закрытые группы) | Приватные пространства, отдельная фаза |
| Управление/репутация | gov.mjs как **Policy-примитив**, обобщённый на Space | Один механизм доступов и для экземпляра, и для пространств |

**Что НЕ берём:** Holochain/серверлесс-с-нуля (новый стек на месяцы), блокчейн с газом
(нет денег и не нужен — единственная задача очерёдности закрывается OpenTimestamps).
Своего протокола не пишем (§4): ценность Nostr в чужой сети, которая на нём говорит.

## Карта «сейчас → цель» по компонентам

| Файл | Сейчас | Цель |
|---|---|---|
| `server.mjs` | реестр+демон, хранит всё | тонкий оракул имён + сидер; роуты личности/контента → события |
| `auth.mjs` | members.json, сессии | проверка событий Identity; членство = событие, не запись сервера |
| `gov.mjs` | доступы/репутация экземпляра | Policy-примитив, scope расширен на `space:<id>` |
| `ws.mjs` | единственная лента | одно реле из пула публичных |
| `ext/*` | рендер CID, window.noet (me/publish/query) | полный SDK примитивов, рендер по типу Space |
| `web/account.js` | редактор страниц | редактор Space/Artifact на SDK |
| `web/people.html` | люди экземпляра | люди + политика любого Space |
| реестр имён | `name→CID` на сервере | `name→координата` как событие-заявка → OTS клиентски |
