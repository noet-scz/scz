# PoC Фазы 1 — IPFS-публикация и честный тест

Цель не «построить реестр», а проверить тех-ядро концепции на чужой инфраструктуре:
страница адресуется по **хешу содержимого** и достаётся из сети, а не с сервера.

## Что в папке

```
poc/
├── bin/ipfs          — kubo v0.41.0 (self-contained, не в системе)
├── ipfs-repo/        — репозиторий узла (IPFS_PATH; в .gitignore)
├── site/index.html   — тестовая статическая страница
├── CIDS.txt          — CID вне IPFS (потеряешь CID → потеряешь вход)
└── README.md         — этот файл
```

## Как запустить узел

```fish
cd /home/nyx/scz/poc
set -x IPFS_PATH /home/nyx/scz/poc/ipfs-repo
./bin/ipfs daemon            # фоновый узел; локальный шлюз на :8080, API :5001
```

Телеметрия kubo отключена (`Plugins.telemetry.Config.Mode off`).

## Опубликовать / обновить страницу

```fish
./bin/ipfs add -rQ --cid-version 1 site     # -> CID директории
./bin/ipfs routing provide <CID>            # анонс провайдера в DHT
```

### ✅ Честный тест «выключи свой узел» (концепция §11) — ПРОЙДЕН

Сначала была иллюзия: публичные шлюзы доставали байты, найдя провайдером **мой**
узел (единственный пиннер) — «всё летает, потому что втихую раздаёт твой же комп».
Чтобы проверить по-настоящему, подняли **второй всегда-включённый узел** (kubo в
Docker на отдельном VPS), который пиннит **тот же CID** (тот же контент → тот же хеш,
сверено байт-в-байт).

Затем — чистая проверка, без публичных шлюзов и без кешей:

1. Основной (мой) узел **выключен**.
2. На ноуте поднят «чистый наблюдатель» — свежий репозиторий, **пустой bootstrap**.
3. Наблюдатель подключён **только** к одному пиру — второму узлу на VPS.
4. `ipfs cat <CID>/index.html` → **998 байт, та же страница**.

Раз единственный источник — VPS, а мой узел был выключен, контент пережил выключение
моей машины. Это и есть «открой с чужой машины» из §11 — по-настоящему.

| Условие | Источник | Итог |
|---|---|---|
| Мой узел ON | мой узел / шлюзы | 200 (но это могла быть иллюзия) |
| Мой узел **OFF**, наблюдатель → только VPS | **второй узел (VPS)** | **998 байт ✅** |

Оперативные детали второго узла (IP, PeerID, как погасить) — в `ops-staging.md`
(не в git). VPS-узел оставлен жить: `--restart unless-stopped`.
