# PoC Фазы 1 — IPFS-публикация и честный тест

Цель не «построить реестр», а проверить тех-ядро концепции на чужой инфраструктуре:
страница адресуется по **хешу содержимого** и достаётся из сети, а не с сервера.

## Что в папке

```
poc/
├── bin/ipfs          — kubo v0.41.0 (self-contained, не в системе)
├── ipfs-repo/        — репозиторий узла (IPFS_PATH; в .gitignore)
├── site/index.html   — тестовая статическая страница
├── CIDS.txt          — CID вне IPFS (потеряешь CID → потеряешь вход)
└── README.md         — этот файл
```

## Как запустить узел

```fish
cd /home/nyx/scz/poc
set -x IPFS_PATH /home/nyx/scz/poc/ipfs-repo
./bin/ipfs daemon            # фоновый узел; локальный шлюз на :8080, API :5001
```

Телеметрия kubo отключена (`Plugins.telemetry.Config.Mode off`).

## Опубликовать / обновить страницу

```fish
./bin/ipfs add -rQ --cid-version 1 site     # -> CID директории
./bin/ipfs routing provide <CID>            # анонс провайдера в DHT
```

## Результаты теста (2026-06-06)

### ✅ Что доказано

| Проверка | Итог |
|---|---|
| Узел поднялся, вошёл в сеть | 35–36 пиров |
| Страница в IPFS, адрес = хеш содержимого | CID v1 получен |
| Локальный шлюз отдаёт | HTTP 200 |
| **Публичный шлюз `ipfs.io`** (не моя машина) | **HTTP 200** |
| **Публичный шлюз `dweb.link`** (не моя машина) | **HTTP 200** |

Вывод: ядро работает. Контент адресуется по содержимому и собирается из сети
сторонней инфраструктурой. **Блокировка по IP против этого бессильна** — единого
сервера нет (это единственный слой, где конструкция честно сильнее обычного сайта,
концепция §9.2).

### ⚠️ Чего тест НЕ доказал (и почему это важно)

**Честный тест «выключи свой узел» (концепция §11) — ещё не пройден.**

Публичные шлюзы достали байты, найдя провайдером **мой** узел — он единственный
пиннит. Это ровно иллюзия из документа: «всё летает, потому что втихую раздаёт твой
же комп». При выключенном узле три свежих шлюза вернули HTTP 000 — но это **не**
чистое доказательство: `cf-ipfs.com` закрыт Cloudflare, прочие могли быть просто
недоступны, а `ipfs.io`/`dweb.link`, раз скачав, сами на время стали провайдерами
из кеша (но кеш не вечен — это не выживание, а отсрочка).

**Настоящий честный тест требует второго постоянного пиннера** — его на одной моей
машине корректно не сымитировать. Это и есть следующий шаг Фазы 1.

## Следующий шаг (storage из роадмапа Фазы 1)

«Хранение — твои 2–3 узла + выписанные CID + бесплатный пиннинг как сетка». Варианты
сделать честный тест проходимым:

- [ ] **Бесплатный пиннинг-сервис** (web3.storage / Pinata free) — нужен аккаунт/API-ключ.
- [ ] **Второй всегда-включённый узел** (старый комп / Raspberry Pi), пиннит тот же CID.
- [ ] После этого: выключить мой основной узел → страница всё ещё открывается = ✅ честный тест.

> Правило документа: без архива (Arweave) не клади в сеть ничего, потеря чего необратима.
> Пока всё бесплатно = хрупко.
